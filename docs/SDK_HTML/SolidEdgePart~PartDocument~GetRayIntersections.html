This value is used to determine if intersections with edges or vertices are entry or exit points. The offset is the distance before and after the intersection location at which it is determined whether the intersecting ray is outside of, inside of or on the body. If the point previous to the intersection (intersection - Offset) is outside of the body and the point after the intersection (intersection + Offset) is inside of or on the body that intersection is classified "seEntry". The opposite case (inside or on the body before and outside after) the intersection point is classified as "seExit". If the ray is outside of the body before and after the intersection point and the intersection point under consideration is the only point of intersection for this ray with this body, it is classified as "seSilhouette". If the intersection point does not lie on the centerline of the intersecting ray, no attempt is made to classify the intersection type. The Offset is not used to classify intersections with faces.
The total number of intersections found between all of the input body objects and all of the defined rays (matching origin and direction sets). The intersections returned are ordered axially for each input ray with each ray's set of intersections followed by the next, in the order the rays were defined.
Index into the input body object array of the body involved in the intersection.
x,y,z coordinates of the location of each intersection result found. If a body is intersected by only the outer part of the defined ray and not its centerline, the returned intersection point will be located on the centerline of the ray and not on the body itself. In this case, the RayIntersectionTypeConstant for the intersection will be set to the value "seTypeUnkown".
x,y,z coordinates of the surface Normal Vector at the point of intersection. This is only applicable if the entity intersected is a face (RayIntersectionEntityConstant = "seFace"), and is set to 0,0,0 for edge and vertex intersections.
Index into the input arrays of ray origins and directions of the ray involved in the intersection.
This optional return value indicates the type of intersection where possible. The possible values are:
o seTypeUnkown: If this value is returned, the type of intersection was indeterminate. This can occur when the body is intersected by the radius of the input ray but not the center line or when the tests for the other intersection types do not pass. This is the default return value.
o seSilhouette: This value is returned when it can be determined that the intersection is the only one for this body and that it is tangential to the outline of the body in the viewing direction.
o seEntry: This value is returned when the ray is outside of the body at the distance of the user-provided offset prior to the intersection and is inside or on the body at the distance of the user-provided offset after the intersection.
o seExit: This value is returned when the ray is inside or on the body at the distance of the user-provided offset prior to the intersection and is outside of the body at the distance of the user-provided offset after the intersection.
Description
Returns intersection points, normal at points, intersection types and intersection information for a set of bodies and set of rays
