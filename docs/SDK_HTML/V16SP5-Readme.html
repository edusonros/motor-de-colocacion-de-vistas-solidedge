------------------------------------
This Update addresses the following:
------------------------------------

=====Assembly=====

PR 1380634: Fixed a crash with undo immediately after edit definition of a Virtual Component.
PR 2043826: Fixed a problem where trying to include would cause a crash.
PR 1362973: Fixed a problem with an assembly relationship
PR 1373381: Fixed a problem with Tee's not getting placed correctly sometimes.
PR 1379885: Fixed a problem not being able to locate reference planes in a part file transferred to a new sub-assembly.
PR 4848111: Fixed a problem with axial constraints using coordinate system.
PR 4932201: Fixed a problem with segments being wrongly discarded as not-connected in the piping command.
PR 5124509: Fixed a problem where sketches or features were marked as being in a warning state
PR 2041353: Fixed a problem where an element was not removed from a rigid set.
PR 5077447: Fixed a problem with dynamic display during the creation of line path.
PR 5126757: Fixed a problem to avoid creating a redundant constraint.
PR 5128151: Fixed a problem with Show All on an unloaded subassembly requiring two invocations.
PR 5124975: Added an option to toggle between V15 and V16 locate behavior in XpresRoute. To set the old locate behavior dbl click on the OldLineSegLocate.reg file delivered in the Solid Edge V16 Program folder. To restore the current locate behavior dbl click on the NewLineSegLocate.reg delivered in the Solid Edge V16 Program folder.

=====Part=====

PR 1341064: Fixed a problem where pre-v15 detail views were losing their style information
PR 5113947: Fixed a 2d solve problem with certain dimension values.
PR 5123343: Fixed a several aborts based on crashlog data.
PR 5128655: Fixed an abort during locate.
PR 5031522: Fixed a scaling problem in Sat translator import.
PR 5117409: Fixed a problem where a loft feature that failed to recompute successfully
PR 1379851: In V16 Update 3 and V16 Update 4, the formatting of the Custom Property value for the exposed variable PhysicalProperties_Density was controlled by the File->File Properties->Advanced Units Density Readout. The default unit readout precision for most Solid Edge files is three decimal places. When the precision for PhysicalProperties_Density falls below the threshold of three decimal places, data can be lost due to formatting the Custom Property text string.This fix reverts to the pre-Update 3 behavior and uses a maximum precision of 11 decimal places to avoid any loss of data. Any part file saved in 16 Update 3 or V16 Update 4 should be reopened and Density checked for correctness. If incorrect, restore the value of Density and save the file in V16 Update 5 or later.

=====Draft=====
 
PR 2045966: Fixed a problem that caused dimensions to tube centerlines to detach on update.
PR 5098719: Fixed a problem with drawing views displaying as empty.
PR 5118204: Fixed a problem with displaying special characters in embedded/linked Word documents.
PR 2045967: Changed show/hide behavior of tube centerline components in drawing views to restore pre-V15 functionality.

=====Insight=====

PR 1356455: "Last modified date" will remain unchanged when non-SE files are opened in Insight Connect.
PR 5118446: Fixed a problem where an file open in SE via RMB was being checked in without saving
PR 5125194: Removed the duplicate document ID search when adding unmanged links to managed files.
PR 5126922: Fixed a problem where the files containing the moved documents open without link resolution errors.
PR 1378793: Fixed a problem where, LCA moves documents several times to released folder and leaves them available
PR 5131463: Fixed a problem where file open is taking a long time when the site and document library name are same.
PR 2044844: Fixed a problem where a managed file containing a part copy of a UG file was flagged as out-of-date when is should not have been.
