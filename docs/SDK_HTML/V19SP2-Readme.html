PR 1549987: Fixed a problem with Unable to open files when SQL Server User has a password containing special characters.
Note: To fix the problem, the SQL Server Username/Password encryption and decryption code was modified to use UnicodeEncoding instead of ASCIIEncoding. The currently stored encrypted Username/Password ( <%WindowsRoot%:>\InsightData\InsightUser.tmp i.e. C:\InsightData\InsightUser.tmp) on the server machine is in the Ascii encoding. So after this fix you will need to run SPDBInfo.exe (Menu: Initialize Insight SQL Server) again to store encrypted SQL Server Username/Password in Unicode encoding otherwise user will get error “InsightUser not present on the SQL Server Content Databases.  Please ask the administrator to add InsightUser on the SQL Server Content Databases.” on the client machine.
